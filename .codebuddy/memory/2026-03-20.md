# 2026-03-20 工作日志

## 新增：任务清单面板 + 底部智能输入框组件

### 需求背景
用户希望将 AI 助手的线性瀑布流对话改为类似 CodeBuddy 的任务清单模式——完成的打勾，进行中有进度条，细节可折叠展开。同时在功能页底部加一个智能输入框，支持不返回首页就继续下指令。

### 新增文件
- `app/task-panel.css` — 任务清单面板 + 底部输入框的通用样式
- `app/task-panel.js` — 通用逻辑组件，包含：
  - `TideoTaskPanel` — 任务清单面板（init/updateTask/autoProgress）
  - `TideoPageInput` — 底部智能输入框（init/show/hide），内置意图识别，可同 Skill 就地操作或切换 Skill 跳转

### 修改文件
- `app/translate-v6.html` — 引入 CSS/JS，右侧面板顶部插入任务清单容器，底部加载初始化脚本
- `app/livestream.html` — 同上，右侧片段面板顶部插入任务清单
- `app/generate.html` — 同上，右侧对话面板顶部插入任务清单
- `app/create.html` — `goToSkill()` 中新增 `sessionStorage.setItem('tideo_tasks', ...)` 传递任务定义

### 设计要点
- 三种任务状态：✅ 已完成（绿色打勾+删除线）、⏳ 进行中（旋转 spinner+进度条）、○ 待执行（灰色空心圆）
- 面板可折叠/展开，顶部显示 `2/5 已完成` 计数
- 底部输入框共享首页的意图识别逻辑，识别到同 Skill 就地执行，识别到不同 Skill 提示切换
- 通过 `sessionStorage` 传递任务定义，功能页读取并渲染

## 重构 v3：任务进度与对话流程精确同步

### 问题
任务进度的 7 步和对话流中的内容对应不上——`autoProgress()` 用定时器自动推进，跟页面实际的对话流程完全无关。对话里还在第 2 步，进度条可能已经跑到第 5 步了。

### 方案
废弃 `autoProgress()` 自动推进机制，改为页面对话流在每个关键节点**主动调用** `TideoTaskPanel.markStep()` 同步进度。

### 修改

**task-panel.js**：
- 调整 `TASK_TEMPLATES`，三个页面的步骤与实际对话流程一一对应：
  - translate: upload → config → erase → subtitle → voice → render（6步）
  - livestream: connect → monitor → detect → clip（4步）
  - generate: parse → textConfirm → style → storyboard → synth → done（6步）
- `init()` 不再自动调用 `autoProgress()`，不再读取 sessionStorage
- 新增 `markStep(taskId, status, detail)` 简洁 API，自动展开面板+显示底部栏
- `autoProgress()` 保留为 fallback，但默认不再使用

**translate-v6.html**：
- `handleFile()` → markStep('upload', 'done') + markStep('config', 'active')
- `showStepPreBubble()` → markStep('config', 'done') + markStep(step, 'active')
- `confirmStep()` → markStep(step, 'done')
- `startProcessing()` → markStep('render', 'active')
- `showResult()` → markStep('render', 'done')

**generate.html**：
- `startParsing()` → markStep('parse', 'active')
- `showTextConfirm()` → markStep('parse', 'done') + markStep('textConfirm', 'active')
- `showStyleConfig()` → markStep('textConfirm', 'done') + markStep('style', 'active')
- `enterStoryboard()` → markStep('style', 'done') + markStep('storyboard', 'active')
- `showStoryboardEditors()` → markStep('storyboard', 'done')
- `startSynth()` → markStep('synth', 'active')
- `enterPlayer()` → markStep('synth', 'done') + markStep('done', 'done')

**livestream.html**：
- `startMonitor()` → markStep('connect', 'done') + markStep('monitor', 'active')
- `addAutoClipSegment()` 首次 → markStep('monitor', 'done') + markStep('detect', 'done') + markStep('clip', 'active')
- `generateClip()` → markStep('clip', 'active') + markStep('detect', 'done')

## 新增：同技能追加任务 + 跨技能跳转携带上下文

### 问题
任务完成后底部输入框输入新指令（如「再翻译成日语」），`tideo:pageinput` 事件只返回随机罐头回复，完全断链——不会启动新任务，也不会做任何有意义的操作。

### 方案架构

**两种场景、两种策略：**
1. **同技能追加**（如翻译完再翻成日语）：在当前页重置进度面板 + 保留视频素材 + 走精简流程
2. **跨技能跳转**（如翻译完想做直播剪辑）：通过 `sessionStorage` 携带素材上下文跳转到目标页

**同技能子意图分类：**
- `new-task`：新语种翻译（重置进度，走全流程）
- `finetune`：配音/字幕样式调整（精调模式）
- `redo`：重做某步骤（部分重跑）
- `chat`：无法识别，给出引导提示

### 修改文件

**task-panel.js：**
- 新增 `SUB_INTENT_KEYWORDS` — 同技能内部子意图关键词映射
- 新增 `TideoTaskPanel.reset(opts)` — 重置进度面板，支持 `skipUpload` 跳过上传步骤
- 新增 `TideoTaskPanel._renderNextSuggestions()` — 任务全部完成后渲染「下一步建议」快捷操作卡片
- 增强 `_handleSubmit()` — 跨技能跳转前调用 `_saveContext()` 保存素材信息
- 新增 `_detectSubIntent(text, skill)` — 同技能子意图检测
- 新增 `_saveContext(fromSkill)` — 跨技能跳转前将视频名/src 存入 `sessionStorage('tideo_carry_context')`

**task-panel.css：**
- 新增 `.task-next-suggestions` / `.task-suggest-chip` / `.task-suggest-chip--cross` 样式

**translate-v6.html：**
- `tideo:pageinput` 监听器完全重写：
  - `action === 'new-task'`：提取目标语言 → 重置进度（跳过上传）→ 模拟完整翻译流程（config→erase→subtitle→voice→render）→ 完成后显示导出按钮
